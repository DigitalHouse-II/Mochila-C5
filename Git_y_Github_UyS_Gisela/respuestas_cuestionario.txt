¿Quién inventó el sistema de control de versión Git y por qué?
Fue creado por Linus Torvalds, desarrollado por él y Junio Hamano.
Se creó para para ser utilizado en el desarrollo del kernel de Linux.

¿A quién pertenece actualmente Github y por qué?
En 2018 Github fue víctima del segundo mayor ataque de denegación de servicio (DDoS) de la historia. Ese mismo año fue comprado por Microsoft. 
En julio de 2020 se hizo pública su hoja de ruta.

¿Hay otra forma que no sea la terminal para trabajar con Github?
la respuesta corta es No. 
Sí podrían crearse los archivos y carpetas con interfaz gráfica (lleva más tiempo que hacerlo directamente desde la terminal), pero luego se necesitará de la terminal para "subir" todo lo creado y/o modificado a nuestro repositorio local, y para posteriormente asociarlo al repositorio online que creemos en Github. De esa manera el contenido puede crearse, compartirse, modificarse y preservarse, haciéndose un seguimiento exacto de cada acción, y permitiendonos volver atrás en nuestro trabajo.
En el caso de Mac o Linux se puede trabajar directamente desde la terminal porque Bash ya está predeterminada en estos. En el caso de Windows necesitamos descargar la aplicación Git Bash que nos va a permitir asociar los repositorios.
